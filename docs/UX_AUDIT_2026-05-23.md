# UX Audit — DrinkX CRM, 2026-05-23

> Scope: walked every route under `apps/web/app/(app)/**`, the lead-card stack (`components/lead-card/**`), pipeline, leads-pool, today widgets, forms, automations, settings, team, triage, audit. Searched for `window.confirm`, `TODO`, `tbd`, `disabled cursor-not-allowed`, orphan files. Looked for dead-end clicks, empty states with no CTA, fake disabled toggles, modal/flow inconsistencies, and missing IA links.

## Executive summary

48 findings: **6 Critical**, **15 Major**, **20 Minor**, **7 Nit**. Codebase is solid overall — most screens have honest empty states and the lead card flow is coherent. The five most-impactful fixes:

1. **`/triage` is invisible to a fresh manager** — the page only renders an `<UnmatchedMessagesSection>` that early-returns `null` when zero items, so the whole route shows just the page title and nothing else. There's nothing telling the user this is the inbox-triage page or how to test it.
2. **Composer's «Файл» mode is a stub that silently records a comment** with `payload_json.note: "upload_tbd"`. The user thinks they uploaded a file; they actually saved a typed description. The composer says so in 11px gray copy but the «Сохранить» button still works.
3. **No `/companies` list page exists** — `companies/[id]` only — yet the global search and lead card link there. There is no way to browse companies from the IA. Sidebar also has no Tasks entry; the only path to `/tasks` is the «Все задачи ↗» link inside one Today widget.
4. **Three orphan duplicate files** are checked in: `ArchiveTab 2.tsx`, `BaseUpdateSection 2.tsx`, `base-update/ConflictCard 2.tsx`. They will compile (no import), but are dead code and confusing.
5. **«Архив» tab on the lead card omits attached files**, but the empty-state copy in TasksTab promises files survive archive. Manager archives a task with files → opens Archive → only sees the task title, no expand for files.

## Severity legend
- **🔴 Critical** — broken flow, action does nothing, dead-end with no escape
- **🟠 Major** — significant UX friction, missing CTA, confusing state
- **🟡 Minor** — polish gaps, tooltip needed, copy could be better
- **🔵 Nit** — cosmetic / not user-facing

## Findings by screen

### `/` (root) — `apps/web/app/page.tsx`
- 🟠 The marketing landing still says `Phase 1 · Foundation`, references a non-existent `AUTOPILOT.md`, and points at the GitHub-Pages prototype as if the production app isn't built yet. Three of its four CTAs (`Войти`, `Today (preview)`, `Pipeline (preview)`) all go into the app — there's nothing introducing what DrinkX CRM is. **File:** `apps/web/app/page.tsx:13-19`. **Fix:** either redirect `/` → `/today` for signed-in users, or replace with a real landing.
- 🟡 `Today (preview)` and `Pipeline (preview)` links bypass auth — if a logged-out user clicks them they hit a protected route. `(app)/layout` will redirect, but the affordance shouldn't pretend to be public.

### `/sign-in` — `apps/web/app/sign-in/page.tsx`
- ✅ **Fixed 2026-07-14.** The «🧪 Войти как тестовый пользователь» button was shipped to production with its credentials hardcoded in `apps/web/app/sign-in/page.tsx` — a backdoor into the CRM for anyone who opened the sign-in page, in a public repo. The button now reads `NEXT_PUBLIC_TEST_LOGIN_*` from a local `.env.local` and is not rendered (nor bundled) without them; the leaked account was retired. The password is not repeated here — this file is public too.
- 🟡 After `signInWithOtp` succeeds, the user gets «Проверь почту» but **no way to start over with a different email** — only «Отправить снова», which resends to the same one. The reset path is the small underlined-text «Отправить снова», but the copy is misleading.
- 🟡 No `/onboarding` route exists. New managers land directly on `/today`, which is fine until you realise there's no first-run guidance, no «set up your pipeline» nudge, no «pick a default workspace» step. Fresh-workspace users see widgets with «0 активных лидов» and no obvious next step.
- 🔵 Google button uses CSS `conic-gradient` to fake the Google G logo. Subtle, but it's a brand asset thing.

## Cross-cutting patterns

### Inconsistent destructive-confirm UX
The codebase has at least four different patterns for «are you sure?»:
1. `window.confirm()` — TemplatesSection, CustomFieldsSection, TasksTab (archive), TaskFilesList, ImportWizard.
2. Inline Yes/No (NotesTab).
