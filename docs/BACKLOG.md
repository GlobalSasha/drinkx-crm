## 1. Вложения файлов к задачам + поиск по файлам  · P1

К задачам (вкладка «Задачи» в карточке лида) добавить прикрепление файлов:
текст, PDF, таблицы, аудио и др. Плюс поиск по этим файлам.

**Контекст из кода:**
- `Activity`-модель уже имеет `file_url` + `file_kind` и тип активности `file` —
  частично готово на уровне данных.
- Не построено: загрузка/хранилище файлов, пайплайн аплоада, лимиты размера/типов.
- Поиск требует решения: по имени файла vs по содержимому (извлечение текста из
  PDF, транскрипция аудио — тяжёлая обработка; STT/LLM уже есть в `inbox`/`enrichment`).

**Открытые вопросы к дизайну:**
1. Где хранить файлы? (Supabase Storage уже подключён для Auth — можно ли его Storage?)
2. Поиск: по именам или по содержимому? Если по содержимому — какие типы индексируем?
3. Лимиты: макс. размер, разрешённые форматы.
4. Привязка: файл к конкретной задаче-Activity или общий пул файлов лида?

> Делать после текущей задачи (вкладка «Задачи» + Today). Брейншторм перед реализацией.

## 3. UI consistency — отложенные части G4  · P2

Из спринта «Odoo-reuse follow-ups + UI consistency» (G4). Безопасные части
сделаны (общий `Empty` на `/team`, warn-level lint-правило `drinkx/no-arbitrary-px`).
Отложено как рискованное / объёмное:

- **Замена arbitrary px на type-токены / шкалу.** ✅ Часть A сделана: 164 точных
  font-size значения (`text-[10px]`→`text-2xs`, `[11px]`→`text-xs`, `[12px]`→`text-sm`,
  `[13px]`→`text-md`, `[28px]`→`text-4xl`) — замены 1:1, размер не меняется. Осталось ~71
  warn: фиксированные ширины/высоты (`w-[180px]`, `h-[148px]`), `backdrop-blur-[Npx]`
  (это радиус блюра, не размер — кандидат на исключение из lint-регэкспа) и 3× `text-[9px]`
  (нет токена). Остаток — легитимные единичные значения, не механический sweep.
- **Распил `LeadCard.tsx`.** ✅ Часть B: шапка (ряды 1–3 + Won/Lost-баннер,
  ~265 строк JSX) вынесена в `LeadCardHeader.tsx` — чистое presentational-извлечение,
  те же классы/DOM, без изменения поведения. Файл 660 → ~334 строк. **НЕ** делал
  оборачивание `leads/[id]` / `companies/[id]` в `PageContainer`: это намеренно
  полноэкранные макеты со своим sticky-заголовком, обёртка ломает вёрстку без пользы.
  Дальнейший распил (вкладки/правая колонка в отдельные секции) — по желанию, маленькими PR.
