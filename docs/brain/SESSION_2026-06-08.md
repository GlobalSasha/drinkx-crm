## Что уехало в прод

### PR #129 — единый дизайн-язык (`ui/unify-page-layout`)
- **Вёрстка 10 экранов** сведена к одной рамке: `PageContainer` + `PageHeader`,
  карточки дизайн-системы; `/incoming` и `/forms` сняты с кастомных sticky-баров.
- **Ширина → fluid-архетипы.** `pageContainerVariants` теперь `surface: data | detail | reading`
  вместо `width`; fluid-база `w-full px-4 sm:px-6 lg:px-8 2xl:px-12` + sanity-cap **1760px**.
  Убрало пустые поля по бокам на широких мониторах.
- **Шрифт → один Manrope** (Inter и Plus Jakarta Sans удалены; у Jakarta нет базовой кириллицы).
  JetBrains Mono — для чисел.
- **Унификация токенов (Фазы 0-6):** одно тёплое семейство `brand-*` (canvas/ink/muted-2/-3 схлопнуты),
  единый радиус `rounded-card` (2rem), карты без тени / оверлеи → `shadow-overlay`,
  тёмные CTA (`bg-ink`) → оранжевый primary (`bg-brand-accent`), мёртвые legacy-токены удалены из конфигов.
- **Фикс:** `rose-600`/`rose-500` → `rose` (в конфиге `rose` — строка без числовой шкалы;
  кнопка удаления формы рендерилась без фона). + равная высота виджетов на `/today`.

### PR #130 — логи (`ops/logging-stack`)
- **Файловое логирование:** structlog + stdlib + uvicorn + трейсбеки → JSON в stdout и в
  ротируемый файл `/app/logs/app-<service>.log` (10MB × 5). dev — читаемый консольный вывод.
  Настройка в `app/observability.py:configure_logging`, вызывается из `create_app` и Celery.
- **`GET /api/admin/logs`** (admin-only, домен `app/logs`): фильтры `level`/`since`/`service`/`contains`.
  Читает те же файлы. Покрыт 4 unit-тестами.
- **Ротация docker-логов** на всех сервисах (`x-logging`) + named volume `drinkx_logs`.
- **`infra/production/logs.sh`** — `errors` / `tail` / `digest` на сервере.
- Проверено в проде: деплой `success`, `/api/admin/logs` → 401 без токена (роут жив).
