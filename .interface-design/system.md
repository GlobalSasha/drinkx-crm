# DrinkX CRM — дизайн-система (эталон)

Извлечено из кода 2026-07-14: `apps/web/lib/design-system.ts` (токены `C`/`T`),
`apps/web/tailwind.config.ts`, `apps/web/app/globals.css` (`.type-*`).
Источник правды — код; этот файл — договорённости и роли, которых код обязан держаться.

## Направление и ощущение

Тёплый, мягкий, «дорогой в тишине» — taste-soft. B2B-инструмент, в котором менеджер
живёт весь день: спокойные кремовые поверхности, один оранжевый акцент, крупные
скругления, минимум линий и теней. Только светлая тема (тёмной нет и не планируется).
Плотность умеренная: base 14px, карточки `p-6`; исключение — виджеты дашборда `/today`
плотнее (`p-5`, сетка `gap-3`) — это осознанное решение, держать единым в пределах экрана.

- **Кто:** менеджер по продажам / руководитель, десятки заходов в день, десктоп + телефон.
- **Задача-глагол:** довести лида до сделки — увидеть, что горит, и сделать следующий шаг.
- **Сигнатура:** пилюльность (все контролы и инпуты — `rounded-full`) на фоне
  сверхкрупных карточных радиусов (2rem); кремовый канвас `#F5F4F0` вместо белого.

### Палитра графиков (решение 2026-07-14)

Категориальный порядок для Chart.tsx и всех графиков — только существующие токены,
новых цветов не вводим:

`chart-1 #FF4E00 (accent) · chart-2 #2B5BA8 (info) · chart-3 #2D7A5A (success) ·
chart-4 #B7672D (warning) · chart-5 #B23A48 (rose) · chart-6 #6B6B6B (muted)`

Добавить в `tailwind.config.ts` как `colors.chart.1..6` (+ CSS-переменные `--chart-1..6`
для JS-конфигов графиков). Один ряд данных = `chart-1`; сравнение «мы/остальные» =
`chart-1` + `brand-panel`.

## Типографика

Один шрифт — **Manrope** (`--font-ui`), моно — **JetBrains Mono** (`T.mono`: ID, email,
ИНН, таймстемпы). Никаких Inter/Roboto/Arial. Иерархия — только готовыми классами
`.type-*` (page-title 28/700, section-title 20/600, card-title 16/600, body 14/400,
label 12/500, caption 12/500/caps, table-header 11/600/caps/wide, kpi-number 24/700/tabular,
kpi-number-lg 32/700/tabular, amount 13/500/tabular, hint 11/500/italic, button 13/600).
Вес и цвет работают раньше размера. Крупные цифры и любые динамические числа —
всегда `tabular-nums` (в `.type-kpi*`/`.type-amount` уже встроено).

## Скругления (решение 2026-07-14 — роли)

| Токен | Значение | Роль |
|---|---|---|
| `rounded-full` | пилюля | кнопки, инпуты, бейджи, чипы, аватары |
| `rounded-md` | **12px — переопределить в конфиге** | мелкие вложенные элементы: пункты меню, миниатюры, внутренние строки-ховеры |
| `rounded-lg` | 24px (уже в конфиге) | вложенные блоки внутри карточек, под-карточки |
| `rounded-xl` | 28px (уже в конфиге) | оверлеи: дропдауны, поповеры, модалки |
| `rounded-card` | 32px | карточки, крупные поверхности |

Сейчас `rounded-md` НЕ переопределён и рендерит чужие 6px (29 мест) — после
переопределения на 12px все они станут мягкими автоматически, заменять поштучно
не нужно (проверить глазами места плотной вложенности). Концентричность: вложенный
радиус ≈ внешний − паддинг, но не меньше 12px.

## Глубина

Стратегия — **границы + сдвиг тона поверхности, карточки без теней**. Единственная
тень — `shadow-overlay` (`0 16px 48px -16px rgba(17,17,17,0.18)`), только для настоящих
оверлеев: модалки, дропдауны, поповеры, тосты. `shadow-sm/md/lg/xl/2xl` и голый
`shadow` — запрещены. Инпуты на белой карточке остаются белыми c границей
(пилюльная форма сама выделяет их).

## Отступы

Сетка **4-8-12-16-24-32** (Tailwind 1/2/3/4/6/8). Карточки `p-6`, layout из `C.layout`
(`max-w-[1800px]`, `px-4 md:px-6`, `pt-[112px]`, `pb-[6vh]` — санкционированные
исключения). Произвольные значения вида `p-[13px]` — запрещены.

## Движение

Только CSS, framer-motion не вводить. Easing — токены `soft`
(`cubic-bezier(0.32,0.72,0,1)`) и `spring` (`cubic-bezier(0.16,1.16,0.3,1)`).
Нажатие — `active:scale-[0.96]` (встроено в `C.button.*`). `transition-all` запрещён —
именованные свойства (`transition-colors`, `transition-transform`, `transition-opacity`).
Длительности < 300ms; частые действия не анимируем. Новое движение — с
`prefers-reduced-motion` фолбэком.

## Компоненты

Сначала `components/ui/*` (Button, Card, Badge, Modal, DataTable, Item, Empty, Kbd,
InlineConfirm, Tabs, Toast, Tooltip…) и токены `C.*` — руками ничего не перекатывать.
Radix-примитивы для стейтфул-контролов уже в проекте. Состояния обязательны:
hover, focus (`C.focusRing`), active, disabled, loading, empty, error.
Touch-таргеты ≥ 44px (coarse-вариант в конфиге для ховер-контролов).

## Числа и таблицы

Числа в колонках — выравнивание вправо + `tabular-nums` (образец: `WorkloadTable.tsx`).
Текст — влево. Заголовки — `.type-table-header`. Пагинация/load-more предпочтительнее
бесконечной прокрутки.
